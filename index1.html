Como Validar Cartões

O algoritmo Luhn (ou "módulo 10") é o padrão da indústria para verificação de números de cartão:

Passo 1: Entenda a estrutura do número do cartão

Um número de cartão de crédito consiste em:

  • BIN (6 primeiros dígitos): Identifica o emissor do cartão
  • Número da conta (dígitos 7-15): Identifica a conta individual
  • Dígito verificador (último dígito): Valida o número usando o algoritmo Luhn

Os cartões têm normalmente 16 dígitos, mas podem variar de 13 a 19 dígitos.

Passo 2: Aplique o Algoritmo de Luhn

O algoritmo Luhn funciona da seguinte forma:

  1. Comece pelo penúltimo dígito e mova para a esquerda, dobrando cada segundo dígito
  2. Se a multiplicação resultar em um número com dois dígitos, some os dígitos (ou subtraia 9)
  3. Some todos os dígitos
  4. Se o total for um múltiplo de 10, o número é válido
function isValidCard(cardNumber) {
cardNumber = cardNumber.replace(/\D/g, '');
if (cardNumber.length < 13 || cardNumber.length > 19) return false;

let sum = 0;
let shouldDouble = false;

for (let i = cardNumber.length - 1; i >= 0; i--) {
let digit = parseInt(cardNumber.charAt(i));
if (shouldDouble) {
digit *= 2;
if (digit > 9) digit -= 9;
}
sum += digit;
shouldDouble = !shouldDouble;
}

return sum % 10 === 0;
}
Passo 3: Identifique a Bandeira do Cartão

Você pode identificar a bandeira pelos primeiros dígitos (BIN):

function detectCardType(cardNumber) {
const firstDigit = cardNumber.charAt(0);
const firstTwoDigits = cardNumber.substring(0, 2);
const firstFourDigits = cardNumber.substring(0, 4);

if (firstDigit === '4') return 'Visa';
if (/^5[1-5]/.test(firstTwoDigits)) return 'Mastercard';
if (/^3[47]/.test(firstTwoDigits)) return 'American Express';
if (/^6011|^64[4-9]|^65/.test(cardNumber)) return 'Discover';
if (/^35(2[8-9]|[3-8][0-9])/.test(firstFourDigits)) return 'JCB';
if (/^3(0[0-5]|[68][0-9])/.test(firstTwoDigits)) return 'Diners Club';
if (/^(4011|4312|4389|4514|4573|4576|5041|5067|5068|5090|5099|6277|6363|650[0-9])/.test(firstFourDigits)) return 'Elo';
return 'Desconhecido';
}

Tecnologia por Trás da Validação

O sistema de validação de cartões envolve várias camadas de segurança e tecnologia:

Algoritmo Luhn

Desenvolvido por Hans Peter Luhn da IBM em 1954, é um simples checksum para verificar erros de digitação.

BIN/IIN

Os primeiros 6 dígitos identificam a instituição emissora (Bank Identification Number/Issuer Identification Number).

PCI DSS

Padrão de segurança que todos os sistemas que processam cartões devem seguir para proteção de dados.

Gateways de Pagamento

Sistemas como Stripe, Pagar.me e PayPal que processam transações de forma segura.

Implementação Completa em JavaScript

// Valida número do cartão com Luhn e verifica a bandeira
function validateCreditCard(cardNumber, expiry, cvv) {
// Validação básica
if (!cardNumber || !expiry || !cvv) {
return { valid: false, message: "Todos os campos são obrigatórios" };
}

// Limpa os dados
const cleanCard = cardNumber.replace(/\D/g, '');
const cleanExpiry = expiry.replace(/\D/g, '');
const cleanCvv = cvv.replace(/\D/g, '');

// Verifica comprimento do cartão
if (cleanCard.length < 13 || cleanCard.length > 19) {
return { valid: false, message: "Número de cartão inválido" };
}

// Verifica validade (MMAA)
if (cleanExpiry.length !== 4) {
return { valid: false, message: "Data de validade inválida" };
}

// Verifica CVV (3-4 dígitos)
if (cleanCvv.length < 3 || cleanCvv.length > 4) {
return { valid: false, message: "CVV inválido" };
}

// Aplica algoritmo Luhn
let sum = 0;
let shouldDouble = false;

for (let i = cleanCard.length - 1; i >= 0; i--) {
let digit = parseInt(cleanCard.charAt(i));
if (shouldDouble) {
digit *= 2;
if (digit > 9) digit -= 9;
}
sum += digit;
shouldDouble = !shouldDouble;
}

const isValid = sum % 10 === 0;
const cardType = detectCardType(cleanCard);

return {
valid: isValid,
type: cardType,
message: isValid ? "Cartão válido" : "Número de cartão inválido"
};
}

Segurança e PCI Compliance

Ao trabalhar com cartões de crédito, é essencial seguir as normas de segurança:

  • Nunca armazene dados sensíveis como números completos de cartão ou CVV
  • Use sempre tokens ou serviços de terceiros para processamento
  • Certifique-se que seu site usa HTTPS
  • Implemente CSP (Content Security Policy) para prevenir ataques XSS
  • Siga as diretrizes do PCI DSS (Payment Card Industry Data Security Standard)
Para aplicações reais, considere usar soluções como Stripe Elements ou Braintree Hosted Fields que cuidam da segurança para você.
